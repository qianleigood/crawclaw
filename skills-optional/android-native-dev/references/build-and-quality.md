# Build, Testing, and Quality Guide

Use this reference when troubleshooting builds, applying M3 quality rules, or planning test coverage.

## Build troubleshooting

Common areas:
- dependency conflicts
- Gradle cache issues
- JVM memory / OOM during build
- resource naming conflicts
- reserved Android names

## Quality priorities

- accessibility
- touch targets
- contrast
- performance
- design consistency

## Testing coverage

Think by layer:
- unit tests
- integration / instrumentation tests
- UI / Compose tests

Run the appropriate build and test commands before handoff.
