# CrawClaw Roadmap

This roadmap is intentionally short. It describes where the project is headed, not a fixed promise that every item will land on a precise date.

## Current priorities

- Stabilize the gateway, sessions, and multi-agent routing surfaces.
- Make onboarding, install, and first-run diagnostics easier to trust.
- Tighten the learning loop across session history, memory recall, skills, and workflows.
- Reduce complexity in `plugin-sdk`, `agents/tools`, and workflow orchestration.
- Improve open-source ergonomics: docs, release hygiene, and contributor workflows.

## Now

- Gateway and session reliability
- Onboarding and install quality
- Memory and recall clarity
- Plugin and channel contract stability
- Open-source docs and contributor setup

## Next

- Explicit skill and workflow promotion paths
- Better web/control-plane end-to-end coverage
- Stronger operator-facing configuration and troubleshooting docs

## Later

- Broader ecosystem packaging polish
- Additional hosted or managed surfaces beyond the local-first core

## How to use this roadmap

- `Now` means active focus and a good place for contributors to start.
- `Next` means the area is on deck once the current stability and documentation work lands.
- `Later` means it matters, but it is not the best place to spend new contributor effort yet.
