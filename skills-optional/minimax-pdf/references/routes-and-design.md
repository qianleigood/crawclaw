# minimax-pdf Routes and Design Guide

Use this reference when deciding between CREATE / FILL / REFORMAT and when visual identity matters.

## Route selection

- **CREATE**: build a polished PDF from scratch
- **FILL**: fill form fields in an existing PDF without changing layout
- **REFORMAT**: parse an existing source document and rebuild it through the CREATE pipeline

## Design-first rule

For CREATE and REFORMAT, read `design/design.md` before choosing visual treatment.

## When to choose this skill

Use it when:
- the final artifact should be PDF-first
- visual quality and print-readiness matter
- the user wants a branded or presentation-quality deliverable

Do not prefer it when the main deliverable should remain editable as DOCX.
